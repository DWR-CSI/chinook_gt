Chinook GT Run Identification Rubric

This document outlines the decision logic used to assign final_call and tributary designations in the rubias.R pipeline.

The process flows sequentially. Below are the steps that evaluate parameters generated from GSI and diagnostic locus models to arrive at the final determinations.

1. Parameters Used

  • Species: Determined by the OkiOts_120255.113 diagnostic marker ("Chinook", "non-Chinook", etc.)
  • fraction_missing: Proportion of missing data (n_miss_loci / (n_miss_loci + n_non_miss_loci))
  • gsi_missing_threshold: Maximum threshold for missing data (parameter args[8])
  • RoSA: Phenotypic run timing prediction from the OTS28 marker ("Early", "Late", "Intermediate", "Uncertain")
  • Prob_repunit: Maximum posterior probability (PofZ) across reporting units
  • PofZ_threshold: Minimum probability threshold for a confident assignment (parameter args[9])
  • repunit: Highest likelihood assigned reporting unit (e.g. "spring", "fall", "latefall", "winter")
  • LFAR_markers_present: True if the markers diagnostic for Late Fall / Fall distinction are present
  • collection: Specific baseline population assignment within the reporting unit

2. Final Call Decision Logic

The logic is evaluated hierarchically. As soon as a condition is satisfied, the respective final_call is assigned and evaluated no further within this pass.

Check Phase Conditions Required Preliminary final_call
High Missing GSI Data
fraction_missing > gsi_missing_threshold + RoSA == "Late" Fall / Late Fall
fraction_missing > gsi_missing_threshold + RoSA == "Early" Spring / Winter
fraction_missing > gsi_missing_threshold Missing Data
Low Confidence Assignment
Prob_repunit < PofZ_threshold + RoSA == "Early" Mixed Spring
Prob_repunit < PofZ_threshold + RoSA == "Late" Mixed Fall / Late Fall
Prob_repunit < PofZ_threshold Mixed
Confident Assignment Overrides
RoSA == "Early" + repunit is "fall" or "latefall" Spring
RoSA == "Late" + repunit == "spring" Fall
fraction_missing < gsi_missing_threshold [Capitalized repunit]
Catch-All
None of the above Assignment Error

2.1 Post-Processing Overrides

After determining the preliminary call, two additional filters run checking for explicit marker findings. This will override the decision above:

  1. Species Override:
    IF Species == "non-Chinook" ➔ THEN final_call becomes non-Chinook
  2. Missing LFAR Markers Override:
    IF LFAR_markers_present == FALSE AND final_call is currently "Fall" or "Latefall" ➔ THEN final_call gets consolidated to Fall / Late Fall

3. Tributary Decision Logic

The tributary call is evaluated independently from final_call using a distinct (but related) set of criteria.

Scenario Condition tributary Outcome
Feather River-Lineage Spring RoSA == "Early" AND
repunit is "fall" or "latefall" AND
Prob_repunit >= PofZ_threshold AND
fraction_missing < gsi_missing_threshold 		Feather River-lineage Spring
Voided Late Run Spring Assigns RoSA == "Late" AND
repunit == "spring" AND
Prob_repunit > PofZ_threshold 		NA (Empty)
Standard Spring Run Collection repunit == "spring" [collection name]
Default Fallback Any other combination NA (Empty)
